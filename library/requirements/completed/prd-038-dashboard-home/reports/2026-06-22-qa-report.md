## Suggestions (consider improving)

- [ ] **Live-stream tail is unlabeled per-harness**, `src/dashboard/web/pages/dashboard.tsx:293-295`, `src/dashboard/web/harness-strip.tsx:9-13`

  AC-5 (index) / c-AC-2 (038c) ask for the stream "labeled/scoped by harness where the record allows." `/api/logs`
  records (`RequestLogRecord`: method/path/status/mode/org) carry no per-line harness field, so the implementation
  correctly renders the tail generically and documents this against the PRD's own 038c-OQ-3 escape hatch
  ("shows the rest as unattributed"). This is **compliant**, not a gap — recorded here only so the follow-up is
  explicit: if PRD-039 / the logger later adds a harness/agent field per request line, revisit `streamLines` to
  scope/label it. No change required for this PRD.

  ```tsx
  // dashboard.tsx:293-295
  // `/api/logs` carries no per-line harness field (c-OQ-3), so it is unlabeled.
  const streamLines = React.useMemo(() => logLines.slice(0, MAX_STREAM_LINES), [logLines]);
  ```

- [ ] **038a Implementation-Note example shows `skills.length` for Team skills; code uses `kpis.teamSkillCount`**, `src/dashboard/web/pages/dashboard.tsx:309`

  The 038a Implementation Notes example tile reads `value={skills.length}`. The code instead binds to
  `kpis.teamSkillCount`. This is the **correct** call: PRD-036c D-2 explicitly prefers the explicit `KpisView.teamSkillCount`
  field over the panel array length precisely "so the KPI does not depend on the panel's array length" and label/number
  "can never desync again." The 038a note predates the authoritative 036c decision; the implementation follows 036c.
  No fix needed — flagged so `library-worker-bee` can reconcile the 038a note text with 036c D-2 if desired.

  ```tsx
  <Kpi label="Team skills" value={kpis.teamSkillCount} accent="dream" />
  ```
