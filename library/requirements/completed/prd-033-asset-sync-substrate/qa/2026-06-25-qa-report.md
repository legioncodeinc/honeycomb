## Suggestions (consider improving)

- [ ] **Stop catalog constants from riding into the hooks bundle**, `src/daemon/runtime/assets/contracts.ts:29,33`

  The security report's Low hygiene finding (confirmed by that report as pre-existing and non-blocking) documents that `SYNCED_ASSET_TYPES` and `SyncedAssetType` are re-exported from `daemon/runtime/assets/contracts.ts`, which transitively pulls `src/daemon/storage/catalog/synced-assets.js`, `catalog/types.js`, `storage/schema.js`, and `storage/sql.js` into the built harness hook bundles. The DeepLake client itself (connection, credentials, transport) does NOT leak and D-6's security purpose is not violated; this is purely a bundle hygiene concern. Relocating the enum to a pure shared module and hardening the invariant test to follow transitive re-exports would remove the issue.

  Suggested remediation: move `SYNCED_ASSET_TYPES`/`SyncedAssetType` to a pure constants file (or define by value in `src/daemon-client/assets/contracts.ts` mirroring how `manifest.ts` mirrors the registry-row shape), then update `tests/daemon/storage/invariant.test.ts` to scan built bundle artifacts for `storage/catalog` symbols rather than relying on static import-specifier text matching.
