## Warnings (should fix)

- [ ] **F-2 nuance: pre-tool VFS recall is the hybrid engine's lexical floor, not full RRF** — `src/daemon/runtime/vfs/api.ts:337-340`, `342-400`

  The pre-tool seam resolves `search` ops through `GET /memory/grep`, which calls the PRD-007 recall collection layer but injects **no embed client**, so the vector channel is skipped and recall runs BM25/ILIKE lexical-only (`degraded: true`). The 075 index describes the intent as reusing "the daemon-side hybrid recall (`recallMemories`, RRF over `<#>` + `ILIKE`)". The always-on 076a arm (`POST /api/memories/recall`) does use the vector channel; the on-demand 075 arm does not. This matches the product's documented degrade-to-lexical posture (never an error, no quality cliff) and `vfs/api.ts` already flags the embed-seam wiring as a follow-up, so it is a Warning, not a blocker.

  ```ts
  // src/daemon/runtime/vfs/api.ts:368
  // No embed client → vector channel skipped → degraded lexical-only (D-4 / a-AC-3).
  ```

  Remediation: wire the embed seam into the VFS grep mount in a follow-up PRD so the on-demand arm matches the always-on arm's semantic depth. Non-blocking.
