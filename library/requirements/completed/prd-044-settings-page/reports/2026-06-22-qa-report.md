## Suggestions (consider improving)

- [ ] **Auth poll lacks visible-tab gating**, `src/dashboard/web/pages/settings.tsx:123-138`

  The `DeeplakeAuthSection` polls `authStatus()` every 5s unconditionally while mounted (plus on `focus`). It is correct and harmless (loopback GET, no secret), but a `document.visibilityState === "hidden"` guard would avoid background polling when the tab is not visible. Not plan-required; the plan only asked for re-read "on focus/poll", which is satisfied.

- [ ] **`recallMemories` engine has redundant-but-welcome recallMode coverage**, `tests/daemon/runtime/memories/recall.test.ts:484-555`

  The recallMode gate is tested at the `recallMemories` engine level here AND at the live HTTP-handler level in `api.test.ts:489-553`. This is defensible defense-in-depth, not a defect. No action required; noted so a future refactor does not delete the engine-level test thinking it duplicates the handler test (they assert different seams: `probe.calls() === 0` proves the embed seam is never touched, which the HTTP test does not).
