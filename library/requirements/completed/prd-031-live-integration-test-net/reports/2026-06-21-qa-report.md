## Would-it-catch-the-bug judgment (the test-net bar)

### AC-1 — route collision → **WOULD FAIL under the bug. GENUINE.**
`tests/daemon/runtime/assembled-net.test.ts:74` boots the app via `assembleTestDaemonApp` → real `assembleDaemon` → `assembleSeams` (every seam in real order behind the real middleware; verified `assemble.ts:457` fires the default seam set, `mountDashboardHost` at `:529`, `mountMemoriesApi` at `:543`). It is **not** an isolated mount.
- `assembled-net.test.ts:99-117` asserts `POST /api/memories/recall` returns **status 200 AND `content-type: application/json` AND NOT `text/html` AND the recall body shape** (`hits[]` containing the fake-storage `memories` hit). A 020b-host-shadows-022 collision would serve the dashboard HTML shell here — the `content-type` + body-shape assertions (not just status) catch that. Load-bearing: `:102-106` + `:110-112`.
- `:138-144` asserts the dashboard host **does** serve `text/html` at its own `/dashboard` path — proving coexistence, not the inverse collision.
- 3/3 AC-1 tests pass in isolation (verified run).

### AC-2 — header gap → **WOULD FAIL under the bug. GENUINE.**
Exercises the **real assembled middleware chain**, not an isolated handler. The runtime-path middleware (`src/daemon/runtime/middleware/runtime-path.ts:262-268`) genuinely rejects a session-group request missing `x-honeycomb-runtime-path` or `x-honeycomb-session` with `400 { error: "bad_request", reason: ... }` **before** any handler runs.
- `assembled-net.test.ts:157-176`: missing `x-honeycomb-session` → 400, `error === "bad_request"`, `reason` matches `/session/i`, **and** `storage.requests` contains **no** `FROM "memories"` query — positive proof the handler never ran.
- `:194-209`: the **same** request **with** the headers → 200 **and** `storage.requests` now **contains** the `memories` arm query — positive proof the handler was reached. This with/without pair is exactly what an isolated mount cannot prove. Load-bearing: `:172-176` (handler-not-reached) + `:204-208` (handler-reached).

### AC-3 — missing-table heal → **WOULD FAIL under the bug. FAITHFUL reproduction, not a dodge.**
This was the highest-scrutiny item. Verdict on each sub-claim:
- **(a) Real engine path.** `tests/integration/missing-table-heal-live.itest.ts:251-254` calls the **real** `recallMemories` (imported from `src/daemon/runtime/memories/index.js`) with `scopedRecallStorage(storage)` injected. The adapter (`:162-170`) rewrites **only** the double-quoted `FROM "<tbl>"` token via `.split(from).join(to)` and forwards every other byte to the live backend. The per-arm tolerance under test — `runArm`'s `isOk(result) ? result.rows : []` (`recall.ts:366-369`) — is the **real engine code**, untouched. The rewrite tokens (`FROM "memories"` / `FROM "memory"` / `FROM "sessions"`) match exactly what the arm builders emit (`recall.ts:195, 213, 231`); embeddings are off (no embed client injected) so only the lexical arms run, so the semantic `FROM` tokens are never emitted — no collateral rewrite. **The adapter cannot make the test pass without the real tolerance absorbing a real `query_error`.**
- **(b) Missing-sibling condition is GENUINE, not assumed.** `:221-232` asserts `tableExists(MEMORIES_TABLE) === true`, `tableExists(MEMORY_TABLE) === false`, `tableExists(SESSIONS_TABLE) === false` **before** driving recall. The siblings are never created (`:96-99` comment + only `MEMORIES_TABLE` seeded at `:209`), so the live backend returns the real missing-relation `query_error` for those two arms.
- **(c) The bar is real.** `:260-276` asserts the `memories` hit IS surfaced (by source+id), recall did NOT throw (reaching the assertions at all proves no 500), `sources === ["memories"]` (siblings degraded to empty, not erroring the read), and `degraded === true` (honest lexical fallback). A control test (`:288-305`) proves an unmatched term still returns clean-empty, never a 500 — separating "no hit" from "the read 500'd."
- Live receipt confirmed by the smoker: `memories=present memory=absent sessions=absent hits=1 sources=[memories] no 500`.
- **If the per-arm tolerance regressed to a single UNION ALL, this test would 500 / wipe the `memories` hit and FAIL.** Faithful.
