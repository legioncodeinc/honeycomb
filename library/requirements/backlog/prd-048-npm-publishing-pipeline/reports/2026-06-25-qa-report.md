## Suggestions (consider improving)

- [ ] **Consider deriving the `"version"` script's `git add` paths from `sync-versions.mjs` rather than re-listing them**, `package.json:51`

  The `"version"` script hard-codes the six manifest paths that `scripts/sync-versions.mjs` already enumerates in `SCALAR_TARGETS` + `MARKETPLACE_PATH` (`scripts/sync-versions.mjs:23-33`). They match exactly today, which is why c-AC-3 passes. But this is a second copy of the same list — the exact drift the script's own header comment warns against ("Single list ... cannot drift from a duplicate copy"). If a harness is added later, a maintainer must remember to update both places or the new manifest silently won't be staged into the version commit. Not in scope for PRD-048 and not blocking; a future hardening could have `sync-versions.mjs` emit the paths (e.g. a `--print-targets` flag) or have the script stage its own writes, so the package.json `git add` cannot fall out of sync.

  ```jsonc
  // package.json:51 — duplicates the list in sync-versions.mjs:23-33
  "version": "node scripts/sync-versions.mjs && git add .claude-plugin/plugin.json harnesses/claude-code/.claude-plugin/plugin.json harnesses/openclaw/openclaw.plugin.json harnesses/openclaw/package.json harnesses/codex/package.json .claude-plugin/marketplace.json",
  ```
