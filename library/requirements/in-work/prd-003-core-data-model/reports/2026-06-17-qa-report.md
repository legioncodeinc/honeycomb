## Suggestions (consider improving)

### S-1: CONVENTIONS.md §3 lists `agents` under the wrong scope category

**File:** `src/daemon/storage/catalog/CONVENTIONS.md:49`

The inline example in the `"agent"` scope bullet reads:

```
memories, sessions, memory, skills, rules, goals, kpis, agents, ontology tables)
```

`agents` appears here, but the actual implementation assigns `scope: "tenant"` to `agents` (carrying `org_id` + `workspace_id`), which is correct per PRD-003e ("These are CROSS-CUTTING / tenancy tables → `scope: "tenant"`"). The code is right; the doc is wrong.

**Risk:** Low. A future Wave Bee reading CONVENTIONS.md alone could apply the wrong scope to a new table that is logically similar to `agents`. The code guard (`defineTable`) doesn't validate scope convention, so the mistake would not be caught automatically.

**Suggested fix:** In CONVENTIONS.md §3, remove `agents` from the `"agent"` scope example and add it to the `"tenant"` scope example alongside `codebase`, `api_keys`, `telemetry`. Adjust the `"tenant"` bullet's example list to:

```
- **`"tenant"`** — cross-cutting table. Carry explicit `org_id` and
  `workspace_id` (`TEXT NOT NULL DEFAULT ''`). (codebase, api_keys, telemetry, agents)
```
