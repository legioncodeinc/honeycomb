## Security close-out history

- **High (rev 1): Cross-tenant BOLA on `/api/sources` + `/api/documents`.** Fixed at `sources/api.ts:84-85` (`getRequestIdentity` cross-check); regression test `tests/daemon/runtime/sources/api.test.ts` (forged org Рєњ 400, matching Рєњ 200). Re-verified green (9/9 in api.test.ts).
- **Critical (rev 2): IPv4-mapped-IPv6 SSRF bypass in the new URL fetcher.** When the W1 fetcher landed, a dotted-decimal-only IP guard let `http://[::ffff:169.254.169.254]/` slip to cloud metadata, because the WHATWG URL parser normalizes the bracketed literal to the hex-hextet form `::ffff:a9fe:a9fe`. `security-worker-bee` added `extractMappedV4` (`url-fetcher.ts:167`), which decodes BOTH the dotted (`::ffff:a.b.c.d`) and the hextet (`::ffff:hhhh:hhhh`) embeddings back to the embedded v4 and re-classifies it (`classifyBlockedV6` calls it at `:202`). Re-verified present and tested: `url-fetcher.test.ts:111-124` (literal-IP block at the API boundary) and `:248-251` (pure-guard hextet regression: `::ffff:7f00:1`, `::ffff:a9fe:a9fe`, `::ffff:0a00:1`, `::ffff:c0a8:101` all blocked), plus `:256-258` proves a v4-mapped PUBLIC address is NOT over-blocked. 44/44 in url-fetcher.test.ts green.
