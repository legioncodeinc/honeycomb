## Suggestions (consider improving)

**S-1 — The `tests/hooks/claude-code/shim.test.ts` c-AC-1 runtime-path sub-assertion uses a slightly loose coupling**

`tests/hooks/claude-code/shim.test.ts:168-177` asserts that the Claude Code shim's runtime-path is `"legacy"` and the Cursor shim's is `"plugin"`. This is correct and meaningful, but the test comment says "Claude Code hook script" which could mislead a future maintainer who extends the shim to use a different path. The assertion itself is sound; a one-line comment clarifying why `"legacy"` is correct for Claude Code would eliminate ambiguity. Severity: Suggestion (no AC impact; the assertion passes and is correct).

**S-2 — The `a-AC-4` test verifies `result.results.map(r => r.harness)` equals `["claude-code","claude-code"]` but this is the harness _name_ not the slug**

`tests/connectors/connector-base.test.ts:189` asserts `result.results.map((r) => r.harness)` equals `["claude-code","claude-code"]`. Both are `ClaudeCodeConnector` instances so the harness property is always `"claude-code"` — this verifies that both detected connectors ran but doesn't verify they were distinct connectors (the `homeA`/`homeB` split). The assertion on `fs.files.has(...)` for both homes at lines 191-192 actually provides the meaningful verification. The `r.harness` assertion is not wrong, just slightly redundant. Consider asserting `result.results.every(r => r.wroteConfig)` instead or in addition. Severity: Suggestion (no AC gap; home-level assertions at lines 191-192 cover the real claim).

**S-3 — The `EXECUTION_LEDGER-prd-019.md` AC matrix e-AC-2/3/6 entries list named tests by count only ("7 named", "3 named") without quoting the test strings**

Unlike every other AC row, the e-AC-2, e-AC-3, and e-AC-6 rows in the ledger don't quote the actual `it()` strings. The tests themselves are correctly named and present — this is a documentation-only gap that makes the traceability table harder to audit in the future without reading the source. The a-AC, b-AC, c-AC, and d-AC rows all quote exact test strings. Severity: Suggestion (docs only, no test gap).
