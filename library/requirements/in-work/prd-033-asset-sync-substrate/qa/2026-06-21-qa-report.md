## Suggestions (consider improving)

- **S-1 (PRD-documented v2 deferral): no content-hash verify before install.** The install side trusts the daemon-returned `native` blob and writes it without re-checking `contentHash` against the bytes. The PRD/ledger explicitly accept this as a v2 note (alongside F-2 segment over-breadth). Not a gap against v1 scope — flagged only so it stays on the v2 radar. Adding a `sha256(native) === asset.contentHash` guard in `install.ts:208-231` before `writeArtifact` would harden against a tampered daemon response.
