## Per-AC evidence

### AC-1 — Deterministic converge unit — **PASS**
`readConverged` (`src/daemon/storage/converge.ts:273-327`) polls `client.query`, tests a
caller predicate, backs off (jittered, capped) and re-polls until the predicate holds or the
budget exhausts. Proven by `converge.test.ts:78-101`: a flapping `ScriptedClient` returns
empty → empty → version-2 (present but too low) → version-3 fresh; the seam converges on the
**4th** poll and returns the fresh `ok` (`expect(result).toBe(fresh)`, `calls.length === 4 < 10`).
Fake clock + fake sleep (`FakeClock`, lines 60-70) make it fast and non-flaky. No live backend.

### AC-2 — Bounded fail-soft — **PASS**
The budget is genuinely bounded on **BOTH** axes:
- **Attempt cap** — `for (attempt = 1; attempt <= maxAttempts; …)` (`converge.ts:302`), proven
  by `converge.test.ts:103-125`: a never-converging client → exactly `maxAttempts` (10) polls,
  returns the LAST real `ok` (`toBe(last)`), and asserts **no fabricated row**
  (`rows.some(r => r.id === "row-1") === false`).
- **Wall-clock** — the next backoff is skipped and `last` returned if it would breach
  `deadline = startedAt + maxWallClockMs` (`converge.ts:315-318`), proven independently by
  `converge.test.ts:127-144`: `maxAttempts:100` but `maxWallClockMs:30` → wall-clock stops it
  "well before" the attempt cap (`calls.length < 100`, `now() <= 30`).
- **Never throws past the union** — `converge.test.ts:146-161`: a `connection_error` result →
  predicate false → budget governs → returns the last non-ok result, no throw, no hang.

### AC-3 — No-flake live read-your-writes — **PASS (by construction)**
`read-converge-live.itest.ts`: N=25 (≥20) loop, each iter `appendVersionBumped` (captures the
`{version}` watermark) → `readConverged` with `watermarkPredicate(id, version)`.
- **Per-iteration HARD assert** (not just an aggregate): line 222
  `expect(sawWrite, …).toBe(true)` fires inside the loop, every iteration; the zero-miss
  aggregate (`expect(misses).toBe(0)`, line 238) is in addition.
- **Bounded fail-soft ghost case is real**: lines 242-263 read a NEVER-written id, assert it

- **Convergence evidence**: a `countingClient` wraps the query so the per-read poll count is
  recorded/printed (`maxPolls`, `flappedReads`); `attempt>1` is direct proof a flap was absorbed.
- Gated (`skipIf(!HONEYCOMB_DEEPLAKE_TOKEN)`), throwaway-isolated (`ci_converge_<runId>`,
  DROP teardown), token read only via `envCredentialProvider`.
- **Smoker confirmed LIVE**: N=25, misses=0. The `maxPolls=1` observation this run is **NOT a
  gap** — see "no-flap run" note below.

### AC-4 — Itests stop hand-rolling it — **PASS (assertion-preservation verified)**
`grep` proof (the 5 PRD-named files only): `controlled-writes-live` and `graph-persist-live`
have **zero** matches for `for (let poll` / `SCAN_POLLS` / `SCAN_DELAY_MS` / `scanDistinct`.
The 3 ontology files retain a `scanRows`/`reduceHighestVersionPerId` helper — **verified NOT a
poll loop**: `scanRows` now delegates the wait to `readConverged(store, sql, s, minRowCount(n))`
and the highest-version-per-id reduction is extracted as a **pure** post-processor over the
converged result. The one surviving `for (let confirm…)` loop in `ontology-deps-live:258-262` is
the genuinely-different D-7 **absence-proof** (prove NO second row appears — the opposite of
poll-until-present), correctly documented as such.

Behavior-preservation note (not a defect): the old `scanDistinct`/`scanRows` UNIONED rows
across all polls to defend against non-monotonic flapping; the new path returns the distinct
set from the single converged poll that first met `minRowCount(expected)`. This is **sound** —
a scan never invents a row, so once one poll returns ≥ `expected` rows that segment IS the
durable truth; the exact-count assertion then holds, and over-counting remains impossible. The
migration's own comments make this reasoning explicit.
