# EXECUTION LEDGER — PRD-033 Asset Sync Substrate

> Orchestrator: `/the-smoker` · Branch: `prd-033-asset-sync-substrate` · SSOT for AC tracking.
> "Dotfiles-over-DeepLake" for harness artifacts: register a skill/agent and control how far it
> propagates via an explicit **tier × style** lattice. `.honeycomb/registry.json` is the SoT;
> a new additive `synced_assets` DeepLake table holds versioned native (+reserved canonical) blobs;
> promotion widens the blast radius, demotion retracts via append-only tombstone rows; a daemon-served
> publish/pull engine installs verbatim native artifacts (identity adapter) with last-writer-wins `.bak`.
> v1 = skills + agents only, native-per-harness, daemon-only DeepLake access (port 3850).

## Open-question decisions (orchestrator, faithful defaults grounded in the PRD notes + existing code — surfaced in the PR)
- **D-1 Device identity:** a generated UUIDv4 persisted at `~/.honeycomb/device.json` (`{device_id, label, createdAt}`), stable per machine, sitting BESIDE the existing `~/.honeycomb/.machine-key` (NOT the raw OS machine-id — privacy + stability). "My devices" is a per-author set the daemon tracks; revoke via CLI writes a device tombstone. (OQ device-identity.)
- **D-2 Skillify manifest:** COEXIST, non-breaking. `.honeycomb/registry.json` is the NEW SoT for substrate-managed assets; the skillify pull manifest (`~/.honeycomb/state/skillify/pull-manifest.json`, PRD-016/018) is left UNTOUCHED. The registry may READ skillify conventions (projectKey, install roots) but does not migrate or delete the manifest in v1. (OQ manifest-migration.)
- **D-3 `honeycomb_id` placement:** stamp into the artifact's YAML frontmatter for skills + agents (both Markdown+YAML, safe across the v1 harnesses); the registry stays AUTHORITATIVE as the fallback. Never stamp where it would break a native parser (registry-only there). (OQ frontmatter-id.)
- **D-4 Retraction UX:** on tombstone/demotion, BACK UP the managed copy to `.bak`, THEN remove it — retraction is REAL (AC-5) while the user's content is preserved via `.bak` (mirrors the last-writer-wins `.bak` discipline). Non-managed sibling files untouched. (OQ delete-vs-unmanaged.)
- **D-5 Append-only + tombstone-as-row:** reuse `appendVersionBumped` (version-bumped INSERT, never UPDATE/DELETE); a tombstone is a row with `tombstone=true`. Live read-backs poll via `readConverged` (DeepLake eventual consistency — never a single immediate read).
- **D-6 Daemon-only DeepLake:** all `synced_assets` reads/writes go through the daemon (port 3850); CLI + hooks never open DeepLake directly.
- **D-7 Native-per-harness + reserved canonical:** store the verbatim native blob keyed `(asset_type, harness)`; ship ONLY the identity adapter in v1 (`parse(render(x))==x`); reserve the `canonical` blob column + the `render/parse` adapter seam so cross-harness render is additive later (no schema change, no re-sync).
