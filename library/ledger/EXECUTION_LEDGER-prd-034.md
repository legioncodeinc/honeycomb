# EXECUTION LEDGER — PRD-034 Resilient Live-Test Strategy

> Orchestrator: `/the-smoker` · Branch: `prd-034-resilient-live-test-strategy` · SSOT for AC tracking.
> Split the live-test strategy by PURPOSE: (034a) an IRL-faithful live suite that asserts the invariants a
> real user depends on — correctness/tenancy/idempotency/no-data-loss/graceful-degradation — with generous/
> absent immediacy bounds + an infra-outage→neutral-skip policy, so the merge gate never reds on DeepLake
> weather; (034b) an on-demand DeepLake stress harness that keeps the strict hammering patterns and turns them
> into a metrics-emitting diagnostic for the vendor. The deterministic suite (PRD-031 plain-CI assembled +
> typecheck/unit/build/audit) stays the hard push gate. NO product/runtime/schema/API change.

## Locked decisions (from the user)
- **D-1** Push gates on the DETERMINISTIC suite only; the IRL-faithful live suite runs nightly + push-SOFT (never merge-blocking).
- **D-2** A sustained infra outage (all-attempts 502/timeout after bounded retries) resolves to a NEUTRAL skip — never a false green, never a hard red attributed to our code.
- **D-3** 034a is a FAITHFUL refactor: keep ALL correctness assertions strict (right value, tenancy isolation, idempotency, no-data-loss-when-healthy, graceful degradation); relax ONLY immediacy assertions (exact-count-now / sub-second visibility / tight per-test timeouts). Reuse `readConverged` for eventually-style waits (no bespoke poll loops). Prove teeth preserved with a deliberately-broken control.
- **D-4** Stress harness FULL scope: `npm run deeplake:stress` + a `workflow_dispatch` CI job, metrics report (per-stmt-kind latency p50/p95/p99/max, error-rate by HTTP status, eventual-consistency convergence-time distribution, throughput-vs-concurrency), concurrency + table-size dials, fixed seed; on-demand ONLY, never gates.
- **D-5** Run the stress harness LIVE once during close-out → walk away with a real DeepLake metrics report.
